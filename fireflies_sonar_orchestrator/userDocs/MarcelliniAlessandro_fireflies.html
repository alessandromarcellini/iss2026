LABORATORIO DI INGEGNERIA DEI SISTEMI SOFTWARE

Requirements

Costruire il simulatore di un sistema costituito da 3 lucciole. Inizialemente, le lucciole lampeggiamo ciascuna con una frequenza (costante) propria.
  • Dopo 10 secondi le lucciole devono lampeggiare in modo sincronizzato;
  • Il sistema è dotato anche di un sonar . Quando il sonar rileva un ostacolo a una distanza minore di una distanza prefissata DMIN, le lucciole devono lampeggiare in modo sincronizzato. Quando la distanza dell’ostacolo (ri)supera DMIN, le lucciole tornano a lampeggiare in modo random

Requirement analysis

Cosa intente il committente per lucciole? Intende entità che lampeggiano ad intervalli variabili. Il programma deve simulare un totale di 3 lucciole.
Le lucciole devono essere rappresentate da dei qactor, che lampeggino inizialmente con periodi casuali e che man mano si sincronizzino allo stesso periodo finale.
Il risultato della sincronizzazione deve essere raggiunto dopo 10 secondi esatti.

Per permetterne la sincronizzazione dopo 10 secondi esistono vari approcci, quello che useremo è l'approccio ad orchestratore.

Scegliamo l'approccio ad orchestratore in quanto semplice da realizzare e facilmente scalabile . Questo introdurrà un single point of failure del sistema e un possibile sovraccarico di messaggi verso l'orchestratore, entrambi problemi facilmente risolvibili tramite replicazione e partitioning dei messaggi.

I messaggi scambiati nel sistema sono definiti come segue: 
Dispatch cellstate : cellstate(X, Y, COLOR) // per il visualizzatore
Dispatch shareFrequency : shareFrequency(Frequency) // permette alle lucciole di inviare la propria frequenza all'orchestratore
Dispatch measureInsufficient : measureInsufficient(s) // inviato dal sonar all'orchestratore per fargli sapere che la distanza registrata ha superato quella attesa
Dispatch measureSufficient : measureSufficient(s) // inviato dal sonar all'orchestratore per fargli sapere che la distanza registrata è minore di quella attesa
Event sync : sync(CurrBestMillis) // emesso dall'orchestratore per dire alle lucciole che è il momento di sincronizzarsi
Event unsync : unsync(unsync) // emesso dall'orchestratore per dire alle lucciole che è il momento di smettere di sincronizzarsi

Il comportamento di una lucciola è rappresentato dal seguente modello: 
QActor firefly context ctxfirefly dynamicOnly { // dynamicOnly significa che questo attore può essere istanziato solo da un altro attore
	[# 
	   var  X          = 0
	   var  Y          = 0
	   var Timer       = java.util.Random().nextLong(2000L,4000L )
	   var OrchestratorTimer = 0L
	   var ExceedingMillis = 0L
	   
	    fun setCellCoords( )  {
     		val coords = name.replace("firefly_","").split("_")   
     		X  = coords[0].toInt()
     		Y  = coords[1].toInt()
  		}		
 	#] 
	
	State s0 initial {
		[# setCellCoords( )                                   #]
		println("$name | X=$X Y=$Y  SharedTimer=$Timer") color magenta
		// Invio della propria frequenza all'orchestratore
		forward fireflies_orchestrator -m shareFrequency : shareFrequency($Timer)
	}
	Transition t0
		whenTimeVar Timer -> flashON
	
	State flashON {
		println("[$name] IND ON") color cyan
	    forward griddisplay -m cellstate : cellstate($X, $Y, 1)
	}
	Transition t0
	    whenTimeVar Timer -> flashOFF
	    whenEvent sync -> changeTimer
	
	State flashOFF {
		println("[$name] IND OFF") color cyan
	    forward griddisplay -m cellstate : cellstate($X, $Y, 0)
	}
	Transition t0
	    whenTimeVar Timer -> flashON
	    whenEvent sync -> changeTimer
	
	// Sync up the fireflies
	State changeTimer {
	    println("[$name] changing timer!") color blue
	    onMsg( sync : sync(CurrBestMillis) ) {
	        [#
	            OrchestratorTimer = payloadArg(0).toString().toLong()
	        #]
	    }
	}
	Goto flashONSync
	
	State flashONSync {
		println("[$name] SYNC ON") color magenta
	    forward griddisplay -m cellstate : cellstate($X, $Y, 1)
	}
	Transition t0
	    whenTimeVar OrchestratorTimer -> flashOFFSync
	    whenEvent unsync -> flashOFF
	
	State flashOFFSync {
		println("[$name] SYNC OFF") color magenta
	    forward griddisplay -m cellstate : cellstate($X, $Y, 0)
	}
	Transition t0
	    whenTimeVar OrchestratorTimer -> flashONSync
	    whenEvent unsync -> flashON
}
Il comportamento dell'orchestratore è definito con il seguente modello: 
QActor fireflies_orchestrator context ctxfirefly {
	[#
		var CurrBestMillis = 0L	
	#]
	
	State s0 initial {
		println("[ORCHESTRATOR] Started") color blue
	}
	Goto orchestrate
	
	State orchestrate {
		// deve ricevere messaggi dalle varie lucciole e stabilire quale è il T
		println("[ORCHESTRATOR] ready to accept messages") color blue
	}
	Transition t0
		whenMsg shareFrequency -> evaluateNewFrequency
		whenMsg measureSufficient -> sync
		whenMsg measureInsufficient -> unsync
	
	State evaluateNewFrequency {
		onMsg( shareFrequency : shareFrequency(Frequency) ) {
			// controlla se la frequenza è migliore ed in caso cambiala
			[#
	            var OuterTimer = payloadArg(0).toString().toLong()
	            println("[ORCHESTRATOR] evaluating new T: $OuterTimer")
	            if (OuterTimer < CurrBestMillis || CurrBestMillis == 0L) {
	            	CurrBestMillis = OuterTimer
	            }
	        #]
		}
	}
	Goto orchestrate
	
	State sync {
		println("[ORCHESTRATOR] EMITTING SYNC") color blue
		emit sync : sync($CurrBestMillis)
	}
	Goto orchestrate
	
	State unsync {
		println("[ORCHESTRATOR] EMITTING UNSYNC") color blue
		emit unsync : unsync(unsync)
	}
	Goto orchestrate
}
Il sonar è invece rappresentato dal seguente modello: 
QActor sonar context ctxfirefly {
	[#
		var  TimerUntilNextToggle = 0L
		
	#]
	State s0 initial {
		println("[SONAR] ready!") color yellow
	}
	Goto measureInsufficient
	
	State measureInsufficient {
		[# TimerUntilNextToggle = java.util.Random().nextLong(10000L,20000L ) #]
		println("[SONAR] Measure Insufficient, next sufficient will be at $TimerUntilNextToggle") color yellow
		forward fireflies_orchestrator -m measureInsufficient : measureInsufficient(s)
	}
	Transition t0
		whenTimeVar TimerUntilNextToggle -> measureSufficient
	
	State measureSufficient {
		[# TimerUntilNextToggle = java.util.Random().nextLong(10000L,20000L ) #]
		println("[SONAR] Measure Sufficient, next insufficient will be at $TimerUntilNextToggle") color yellow
		forward fireflies_orchestrator -m measureSufficient : measureSufficient(s)
	}
	Transition t0
		whenTimeVar TimerUntilNextToggle -> measureInsufficient
}
