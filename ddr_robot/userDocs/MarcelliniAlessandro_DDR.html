LABORATORIO DI INGEGNERIA DEI SISTEMI SOFTWARE

Requirements

Costruire un sistema software che comanda un Differential Drive Robot (DDR) in modo che,
partendo dalla posizione iniziale HOME, il robot si sposti lungo il perimetro di una stanza rettangolare vuota.
Il DDR si deve fermare quando tornato in HOME.

Il DDR è anche sensibile ai dati rilevati da un Radar.
Quando il Radar rileva un 'intruso' a una distanza minore di una distanza prefissata ``DMIN``:
  • se il DDR si sta muovendo , il DDR si ferma fino a che il Radar non rileva più un 'intruso' così vicino;
  • se il DDR è fermo in HOME, il DDR ruota su sè stesso fino a che un 'intruso' così vicino scompare.

Requirement analysis

Che cosa intende il committente per Differential Drive Robot (DDR)?
Un DDR è un tipo di robot mobile il cui movimento si basa su due ruote azionate separatamente posizionate su entrambi i lati del corpo del robot. Tramite tali ruote quindi esso può cambiare direzione variando la velocità relativa di rotazione delle sue ruote, non richiedendo un ulteriore movimento di sterzata.
I requisiti dicono di modellare un sistema che controlli il robot, questo implica che il robot non è un'entità proattiva, bensì reattiva agli stimoli indotti dal controllore. Pertanto il robot non è altro che un servizio messo a disposizione del controllore. Per modellare il comportamento del robot utilizziamo il linguaggio QAK che ci permette di modellare in maniera formale entità come attori, riducendo l'abstraction gap della mera implementazione.
Modello ddr: 

Event rotate : rotate(Degrees)
Event move : move(Distance)

QActor robot context ctxddr {
	[#
		var BodyRadius = 2.0
		
		var CurrX = 0.0 // posizione in X
		var CurrY = 0.0 // posizione in Y
		var CurrRotationDegree = 0.0 // da 0 a 359.0
		
	#]
	State s0 initial {
		println("[DDR] ready!") color cyan
	}
	Goto waitCommand
	
	State waitCommand {
		println("[DDR] waiting for commands!") color cyan
	}
	Transition t0
		whenEvent rotate -> rotate
		whenEvent move -> move
	
	State rotate {
		onMsg(rotate : rotate(Degrees)) {
			[#
				var ReceivedDegrees = payloadArg(0).toString().toDouble()
				
				CurrRotationDegree = (CurrRotationDegree + ReceivedDegrees) % 360
			#]
			println("[DDR] rotated of $ReceivedDegrees") color cyan
		}
	}
	Goto waitCommand
	
	State move {
		onMsg(move : move(Distance)) {
			[#
				var ReceivedDistance = payloadArg(0).toString().toDouble()
				
				// converti gradi in radianti
				val radians = Math.toRadians(CurrRotationDegree)
				
				// aggiorna posizione
				CurrX += ReceivedDistance * kotlin.math.cos(radians)
				CurrY += ReceivedDistance * kotlin.math.sin(radians)
			#]
			println("[DDR] moved of $ReceivedDistance, new current position: ($CurrX, $CurrY)") color cyan
		}
		
	}
	Goto waitCommand
}


Che cosa intende il committente per Stanza?
La stanza è lo spazio in cui il robot può muoversi, essa da specifica è definita come rettangolare e, quindi dotata di 4 lati in cui ogni coppia di lati paralleli ha stessa lunghezza.
Pertanto per modellare la stanza useremo una coppia di variabili reali che indichino la lunghezza della base e dell'altezza della stanza (in unità robotiche): 

var StanzaB = 10.0L
var StanzaH = 15.0L


Che cosa intende il committente per HOME?
Posizione specifica all'interno della stanza da cui parte il robot inizialmente, rappresentabile come un set di coordinate e area occupata dal robot (in quanto non puntiforme). 

var HomeX = 0.0L
var HomeY = 0.0L
var HomeR = 2.0L // raggio del robot (circolare)


Che cosa intende il committente per radar?
Il radar è un dispositivo che misura la distanza dall'oggetto davanti ad esso con una propria frequenza di campionamento, notificandola ai dispositivi che da esso dipendono. 

Event currMeasure : currMeasure(CurrDistance)

QActor sonar context ctxddr {
	[#
		var  Timer = 0L // in ms
		var  CurrDistance = 0.0L
		
	#]
	State s0 initial {
		println("[SONAR] ready!") color yellow
	}
	Goto measure
	
	State measure {
		[# CurrDistance = java.util.Random().nextDouble(0.0, 20.0) #] // campione distanza corrente
		println("[SONAR] Current measure: $CurrDistance") color yellow
		emit currMeasure : currMeasure($CurrDistance) // emissione evento a chi interessa con la misura
	}
	Transition t0
		whenTimeVar Timer -> measure // ripeti al prossimo periodo
}
