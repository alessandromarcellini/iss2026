LABORATORIO DI INGEGNERIA DEI SISTEMI SOFTWARE

Requirements

Costruire il simulatore di un sistema costituito da 3 lucciole. Inizialemente, le lucciole lampeggiamo ciascuna con una frequenza (costante) propria.
  • Dopo 10 secondi le lucciole devono lampeggiare in modo sincronizzato;
  • Il sistema è dotato anche di un sonar . Quando il sonar rileva un ostacolo a una distanza minore di una distanza prefissata DMIN, le lucciole devono lampeggiare in modo sincronizzato. Quando la distanza dell’ostacolo (ri)supera DMIN, le lucciole tornano a lampeggiare in modo random

Requirement analysis

Cosa intente il committente per lucciole? Intende entità che lampeggiano ad intervalli variabili. Il programma deve simulare un totale di 3 lucciole.
Le lucciole devono essere rappresentate da dei qactor, che lampeggino inizialmente con periodi casuali e che si sincronizzino se necessario.
Il risultato della sincronizzazione deve essere raggiunto quando la distanza misurata dal sonar è inferiore a DMIN.

Per permetterne la sincronizzazione tra le lucciole esistono vari approcci, quello che useremo è l'approccio a coreografia.

Scegliamo l'approccio a coreografia in quanto seppur complesso da realizzare è anche il più attinente alla realtà e, soprattutto, permette di avere un sistema fault tollerant, senza single point of failure..

I messaggi scambiati nel sistema sono definiti come segue: 
Event globalStart : globalStart(start) // emitted from creator in order to make all the fireflies start at the same time
Dispatch cellstate : cellstate(X, Y, COLOR) // forwarded from firefly to the visualizer
Dispatch synch_up : synch_up(synch) // autodispatch to start synching with the others
Event outerFlash : outerFlash(T) // emitted from firefly to notify the others that the firefly has blinked
Event measureInsufficient : measureInsufficient(s) // emitted from sonar to make the fiflies unsync
Event measureSufficient : measureSufficient(s) // emitted from sonar to make the fiflies sync

Il comportamento di una lucciola è rappresentato dal seguente qactor: 
QActor firefly context ctxfirefly dynamicOnly {
    [#
       var X = 0
       var Y = 0
       var Timer = java.util.Random().nextLong(2000L,4000L)
       var SyncTimer = Timer

	    fun setCellCoords( )  {
     		val coords = name.replace("firefly_","").split("_")   
     		X  = coords[0].toInt()
     		Y  = coords[1].toInt()
  		}
    #]

    State s0 initial {
        [#
        	setCellCoords()
         #]
        println("$name | coord=($X,0) Timer=$Timer") color magenta
    }
    Transition t0
    	whenEvent globalStart -> flashON

    // INDEPENDENT BLINKING
    State flashON {
        println("$name | [IND] ON") color cyan
        forward griddisplay -m cellstate : cellstate($X, 0, 1)
    }
    Transition t0
        whenTimeVar Timer -> flashOFF
    	whenEvent measureSufficient -> startSync

    State flashOFF {
        println("$name | [IND] OFF") color cyan
        forward griddisplay -m cellstate : cellstate($X, 0, 0)
    }
    Transition t0
        whenTimeVar Timer -> flashON
        whenEvent measureSufficient -> startSync

    // TRANSITION TO SYNCH MODE ------------------------------------------------------------------------------------
    State startSync {
        println("$name | entering SYNC mode") color blue
        // flash ON and broadcast our timer to all peers
        forward griddisplay -m cellstate : cellstate($X, 0, 1)
        emit outerFlash : outerFlash($Timer)
    }
    Transition t0
        whenTimeVar Timer -> flashOFFSync

    // SYNCED BLINKING ----------------------------------------------------------------------------------------------
    State flashONSync {
        println("$name | [SYNC] ON  Timer=$Timer") color magenta
        forward griddisplay -m cellstate : cellstate($X, 0, 1)
        emit outerFlash : outerFlash($Timer)
    }
    Transition t0
        whenTimeVar SyncTimer -> flashOFFSync
        whenEvent measureInsufficient -> flashOFF

    State flashOFFSync {
        println("$name | [SYNC] OFF Timer=$Timer") color magenta
        forward griddisplay -m cellstate : cellstate($X, 0, 0)
    }
    Transition t0
        whenTimeVar SyncTimer -> flashONSync
        whenEvent outerFlash -> absorbWhileOFF
        whenEvent measureInsufficient -> flashON

    State absorbWhileOFF {
    	println("[$name] absorbing") color yellow
        onMsg( outerFlash : outerFlash(T) ) {
            [#
                val incomingTimer = payloadArg(0).toString().toLong()
                println("$incomingTimer < $SyncTimer ?")
                if (incomingTimer < SyncTimer) {
                    SyncTimer = incomingTimer
                    println("$name | adopted faster timer: $SyncTimer")
                }
            #]
        }
    }
    Goto flashONSync
}
Mentre il sonar è rappresentato dal seguente modello: 
QActor sonar context ctxfirefly {
	[#
		var  TimerUntilNextToggle = 0L
		
	#]
	State s0 initial {
		println("[SONAR] ready!") color yellow
	}
	Goto measureInsufficient
	
	State measureInsufficient {
		[# TimerUntilNextToggle = java.util.Random().nextLong(10000L,20000L ) #]
		println("[SONAR] Measure Insufficient, next sufficient will be at $TimerUntilNextToggle") color yellow
		emit measureInsufficient : measureInsufficient(s)
	}
	Transition t0
		whenTimeVar TimerUntilNextToggle -> measureSufficient
	
	State measureSufficient {
		[# TimerUntilNextToggle = java.util.Random().nextLong(10000L,20000L ) #]
		println("[SONAR] Measure Sufficient, next insufficient will be at $TimerUntilNextToggle") color yellow
		emit measureSufficient : measureSufficient(s)
	}
	Transition t0
		whenTimeVar TimerUntilNextToggle -> measureInsufficient
}
